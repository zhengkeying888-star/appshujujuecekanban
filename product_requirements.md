# APP 商业化决策看板 — 产品需求文档（PRD）v4.0

> 文档版本：v4.0
> 更新日期：2026-05-15
> 框架版本：v2.1
> 目标用户：运营团队、管理层、产品侧
> 数据周期：2026 年 3–4 月
> 交付形式：单文件 HTML BI 看板（`dashboard/v2/index.html`），支持一键导出独立 HTML 月报

---

## 1. 项目背景与目标

基于 `APP线索广告位拆解 3-4月更新版.xlsx`（约 3.6 万行、37 列明细数据）、`3-4月月活人数.xlsx`（用户等级月活数据）及 `APP品类流量结构.csv`（品类级月度汇总），构建可本地运行的单文件 HTML BI 看板。

**核心诉求转变**：从「完课率-centric」转向「APP产值归因-centric」。只有跟 APP 产值强相关的指标才是核心指标。复盘目的是向上汇报，需要输出结论、支持交互下钻、假设分析。

**业务目标**：
1. 让管理层 30 秒看懂「上个月 APP 产值怎么了」（一页纸诊断）
2. 区分「前端流量入口效率」和「后端变现效率」，避免把 CTR 下滑和产值下滑混为一谈
3. 支持下钻到「资源位 × 价格带 × 品类类型」三维交叉，定位产值下降的具体病灶
4. 基于下钻结果，自动生成可落地的策略建议与行动清单

## 3. 核心指标重新定义

从数据出发，APP 产值 = 各资源位带来的 GMV 总和。

拆解公式：
```
资源位 GMV = 曝光UV × CTR × CTCVR × 首单转化率 × 客单价
```

但复盘时更关注**归因视角**：
- 总 GMV 环比下降，是**哪个资源位**拖了后腿？
- 该资源位产值下降，是因为**线索量少了**，还是**单线索产出的钱少了**？
- 价格带结构是否恶化（0元课占比上升 → 整体 ARPU 被拉低）？

因此，核心指标应聚焦在：

| 层级 | 指标 | 用途 | 公式 |
|------|------|------|------|
| **产值结果** | GMV、线索数、首单数 | 结果呈现 | `SUM(首单流水)` / `SUM(线索数)` / `SUM(首单数)` |
| **后端效率** | 首单转化率（线索→首单）、单线索产出（LTV） | 产值归因核心 | `SUM(首单数) / SUM(线索数)` / `SUM(首单流水) / SUM(线索数)` |
| **前端效率** | CTR（点击/曝光）、CTCVR（领课转化率=线索/点击）、线索生成率（线索/曝光） | 流量入口效率 | `点击UV / 曝光UV` / `线索数 / 点击UV` / `线索数 / 曝光UV` |
| **结构健康** | 价格带占比（0元/1.1元/3.9元/其他）、品类类型占比（正式品/孵化品） | 结构诊断 | 该价格带线索数 / 当月总线索数 |

**完课率、到课率、好友率属于后链路过程指标**，可以在下钻时展示，但不应作为 headline 级别的核心诊断信号。

### 3.1 指标口径汇总

| 指标名称 | 计算公式 | 说明 |
|---------|---------|------|
| **CTR** | `点击UV / 曝光UV` | 前端流量吸引效率 |
| **CTCVR（领课转化率/点击转化率）** | `线索数 / 点击UV` | 点击→线索的转化效率 |
| **线索生成率** | `线索数 / 曝光UV` | 综合入口效率 |
| **首单转化率（线索→首单）** | `SUM(首单数) / SUM(线索数)` | **唯一正确公式，严禁直接 AVG 原始 `转化率` 字段** |
| **完课→首单转化率** | `首单数 / 完课数` | 后链路过程指标 |
| **单线索产出（LTV）** | `GMV / 线索数` | 单个线索带来的产值 |
| **单曝光产出** | `GMV / 曝光UV` | 每次曝光的产值贡献 |
| **单点击产出** | `GMV / 点击UV` | 每次点击的产值贡献 |
| **环比** | `(本月值 - 上月值) / 上月值 × 100%` | 上月值为 0 时显示「—」 |

## 4. 四屏框架（v2.1）

看板采用四屏叙事结构，每屏对应一个决策层级：

| 屏幕 | 标题 | 决策层级 | 目标 |
|-----|------|---------|------|
| 屏幕1 | 核心产值诊断 | 「发生了什么」 | 让管理层 30 秒看懂上个月 APP 产值变化 |
| 屏幕2 | 转化链路全景 | 「哪里坏了」 | 区分前端流量效率 vs 后端变现效率 |
| 屏幕3 | 根因下钻 | 「为什么坏了」 | 三维交叉定位产值下降的具体病灶 |
| 屏幕4 | 策略建议 | 「怎么办」 | 自动生成可落地的策略建议与行动清单 |

### 4.1 屏幕1：核心产值诊断

**目标**：让管理层 30 秒看懂「上个月 APP 产值怎么了」。

**布局**：
- **Headline**：基于真实数据的自动诊断
  - 示例：`4月 GMV 环比下降 10.9%（-¥23.7万），主因：选课中心产值下滑 ¥14.9万 + 学习页产值下滑 ¥10.3万`
  - 归因逻辑：GMV 变化最大的 Top 2 资源位作为 headline 主因
- **KPI 卡片（4 张，产值强相关）**：
  1. GMV（3月 → 4月，环比）
  2. 线索数（3月 → 4月，环比）
  3. 整体首单转化率（线索→首单，环比）
  4. 单线索产出（LTV，环比）
- **信号卡片（3 张）**：基于数据自动生成的关键变化信号
  - 例：`学习页单线索产出：¥84.8 → ¥56.4（↓33%），价格带结构恶化，0元课占比上升`
- **行动建议卡片（3 张）**：按负责方（产研/运营/内容）分类的 P0 行动项
- **次月行动清单**：P0/P1/P2 优先级行动卡片

**删除**：完课率、到课率 headline 信号卡片。

### 4.2 屏幕2：转化链路全景

**目标**：区分「前端流量入口效率」和「后端变现效率」，避免把 CTR 下滑和产值下滑混为一谈。

**布局**：左右分栏

**左侧：前端链路效率（曝光 → 线索）**
- 各资源位「CTR」对比柱状图（3月 vs 4月）
- 各资源位「CTCVR = 领课转化率 = 线索/点击」对比柱状图
- 各资源位「线索生成率」对比（线索/曝光）
- 作用：回答「是不是流量入口坏了」「哪个资源位的点击→线索效率在恶化」

**右侧：后端产值效率（线索 → GMV）**
- 各资源位「首单转化率」对比柱状图（3月 vs 4月）
- 各资源位「单线索产出（LTV）」对比柱状图
- 作用：回答「哪个资源位的线索不值钱了」

**下方：全宽诊断卡片（3 张）**
- 基于真实数据，自动找出变化最大的 3 个环节

### 4.3 屏幕3：根因下钻

**目标**：支持下钻到「资源位 × 价格带 × 品类类型」三维交叉，定位产值下降的具体病灶。

**布局**：

**模块1：资源位产值矩阵（全宽散点图）**
- 横轴：线索数（对数刻度）
- 纵轴：单线索产出（GMV/线索数）
- 气泡大小：GMV 规模
- 四象限：右上（明星）、左上（精准可扩量）、右下（粗放需调结构）、左下（观察淘汰）
- 点击交互：选中资源位，联动下方模块

**模块2：资源位 GMV 瀑布图（半宽）**
- 展示各资源位 3月→4月 的 GMV 变化量
- 直观看出「谁拖了后腿、谁补了缺口」

**模块3：选中资源位的价格带结构（半宽）**
- 默认展示整体，点击模块1 的某个点后，切换为该资源位的价格带饼图
- 3月 vs 4月 并排环形图
- 标注各价格带占比变化（pp）

**模块4：价格带全局结构（1/3 宽）**
- 3月 vs 4月 双环形图（真实数据）
- 点击效率矩阵联动：切换为选中资源位的价格带结构

**模块5：品类类型效率（1/3 宽）**
- 正式品 vs 孵化品 的线索数、转化率、GMV 对比
- 基于 `category_name` 与 CSV `品类` 的映射
- 点击效率矩阵联动：切换为选中资源位的品类类型结构

**模块6：用户等级效率（1/3 宽）**
- 横向堆叠柱状图（真实数据）
- 展示该资源位带来的线索分别来自哪些用户等级

### 4.4 屏幕4：策略建议

**目标**：基于屏幕3的下钻结果，自动生成可落地的策略建议，支持一键导出为汇报材料。

**用户明确：不需要「预期 ROI」模拟调配。**

**布局**：

**模块A：核心结论卡片（全宽，3 张）**
- 基于数据自动生成的 3 条核心结论（替代虚构的策略卡片）
- 结论生成逻辑示例：
  - 若某资源位线索量大但单线索产出低于均值 30% → 「XX 资源位线索质量偏低，建议排查价格带结构」
  - 若 0元课占比上升 > 5pp → 「低价引流结构加重，需关注高价值转化」
  - 若某资源位 GMV 环比下降贡献最大 → 「XX 资源位是产值下降主因，需专项复盘」

**模块B：资源位效率预警表（全宽）**
- 列：资源位 | 线索数环比 | 首单转化率环比 | 单线索产出环比 | GMV 贡献变化 | 健康状态
- 健康状态逻辑：
  - 若 CVR 环比下滑 ≥ 10% 或 单线索产出环比下滑 ≥ 15% → 「预警」（黄）
  - 若 GMV 贡献为正且 CVR 稳定 → 「健康」（绿）
  - 其余 → 「观察」（灰）
- 支持点击行下钻到该资源位的历史趋势

**模块C：次月核心行动清单（全宽）**
- 基于模块A 的结论，自动生成行动项（人工可编辑）
- 格式：序号 + 行动标题 + 负责方 + 截止日期 + 优先级
- 数据来源：`action_checklist` 由 Python 根据数据自动触发规则生成，而非硬编码

## 5. 交互逻辑

### 5.1 月份切换（顶部导航）

三种互斥视图：3月 / 4月 / 3-4月对比（默认）

- **KPI 卡片**：展示所选月份数值；对比视图展示 4 月数值 + 环比
- **趋势图/柱状图**：根据 `currentMonth` 切换数据源
- **表格**：通过 CSS `display: none` 隐藏非当前月份列
- **图表**：`renderCharts()` 根据 `currentMonth` 重绘

### 5.2 效率矩阵点击联动（屏幕3）

- 点击散点图中某个资源位气泡：
  1. 高亮该资源位，其他资源位变暗
  2. 价格带饼图切换为该资源位的价格带结构
  3. 旭日图（价格带×品类类型）切换为该资源位的结构
  4. 用户等级堆叠图切换为该资源位的用户等级分布
  5. TOP10 表格高亮对应行
- 再次点击同一资源位或点击空白画布：重置所有高亮与联动

### 5.3 导出 HTML 月报

- 点击顶部「导出 HTML 月报」按钮
- 将当前视图所有 ECharts 图表转为 PNG 静态图
- 移除交互元素（按钮、导航、折叠状态）
- 生成独立 HTML 文件下载

## 7. 技术实现

### 7.1 数据流

```
generate_analysis.py
  → 读取 Excel + CSV
  → 清洗、过滤合计行、计算指标
  → 输出 data_analysis_output.json

generate_dashboard_v2.py
  → 读取 data_analysis_output.json
  → Python f-string 拼接 HTML（含内嵌 CSS + JS + RAW_DATA）
  → 输出 dashboard/v2/index.html
```

**每次数据或模板改动后，必须依次运行两个脚本。**

### 7.2 关键技术约束

- **单文件 HTML**：Tailwind CSS CDN + ECharts 5.5 + ECharts WordCloud 2.1.0
- **f-string 转义陷阱**：严禁在 f-string 内使用 `\n`、`\'`、`\"` 等反斜杠转义序列；JS 对象字面量中的 `{` `}` 必须转义为 `{{` `}}`
- **转化率口径**：唯一正确公式 `SUM(首单数) / SUM(线索数)`，严禁直接 AVG 原始 `转化率` 字段
- **合计行过滤**：必须在任何计算前执行 `df = df[df['stat_month'] != '合计']`
