# APP 商业化决策看板 v2.1 — 框架重构方案

## 用户核心反馈

1. **完课率不是核心指标**，只有跟 APP 产值强相关的才是。
2. **CTR 是 APP 商业化页面指标**，不是后链路；要区分「前端页面效率」和「后端产值效率」。
3. **预期 ROI 不是相关指标**，资源位调整表不需要模拟调配 ROI。
4. **复盘目的是向上汇报**，需要输出结论、支持交互下钻、假设分析。

---

## 核心指标重新定义：什么是「APP 产值归因的效率」

从数据出发，APP 产值 = 各资源位带来的 GMV 总和。

拆解公式：
```
资源位 GMV = 曝光 UV × CTR × 线索生成率 × 首单转化率 × 客单价（ARPU）
```

但复盘时更关注**归因视角**：
- 总 GMV 环比下降 10.9%，是**哪个资源位**拖了后腿？
- 该资源位产值下降，是因为**线索量少了**，还是**单线索产出的钱少了**？
- 价格带结构是否恶化（0元课占比上升 → 整体 ARPU 被拉低）？

因此，核心指标应聚焦在：

| 层级 | 指标 | 用途 |
|------|------|------|
| **产值结果** | GMV、线索数、首单数 | 结果呈现 |
| **后端效率** | 首单转化率（首单数/线索数）、单线索产出（GMV/线索数） | 产值归因核心 |
| **前端效率** | CTR（点击UV/曝光UV）、CTCVR=点击转化率=领课转化率（线索数/点击UV）、线索生成率（线索数/曝光UV） | 流量入口效率 |
| **结构健康** | 价格带占比（0元/1.1元/3.9元/其他）、品类类型占比（正式品/孵化品） | 结构诊断 |

**完课率、到课率属于后链路过程指标**，可以在下钻时展示，但不应作为 headline 级别的核心诊断信号。

## 四屏框架重构（v2.1）

### 屏幕1：核心产值诊断（向上汇报的「一页纸」）

**目标**：让管理层 30 秒看懂「上个月 APP 产值怎么了」。

**布局**：
- **Headline**：基于真实数据的自动诊断（而非虚构的「完课率下滑 7pp」）
  - 示例（基于真实数据）：「4月 GMV 环比下降 10.9%（-¥23.7万），主因：选课中心产值下滑 ¥14.9万 + 学习页产值下滑 ¥10.3万」
- **KPI 卡片（4 张，产值强相关）**：
  1. GMV（3月 ¥217.3万 → 4月 ¥193.5万，环比 -10.9%）
  2. 线索数（16448 → 16112，环比 -2.0%）
  3. 整体首单转化率（7.65% → 7.06%，环比 -7.7%）
  4. 单线索产出（¥132.1 → ¥120.1，环比 -9.1%）
- **GMV 变化归因条形图**：各资源位对总 GMV 环比变化的贡献值（瀑布图/条形图）
  - 正向：首页弹窗 +¥3.6万、个人主页 +¥1.1万
  - 负向：选课中心 -¥14.9万、学习页 -¥10.3万、课程 banner -¥2.5万

**删除**：完课率、到课率信号卡片。

---

### 屏幕2：前端效率 vs 后端产值（双视角对比）

**目标**：区分「前端流量入口效率」和「后端变现效率」，避免把 CTR 下滑和产值下滑混为一谈。

**布局**：左右分栏

**左侧：前端链路效率（曝光 → 线索）**
- 各资源位「CTR」对比柱状图（3月 vs 4月）
- 各资源位「CTCVR = 点击转化率 = 领课转化率」对比柱状图（线索数/点击UV）
- 各资源位「线索生成率」对比（线索数/曝光UV，作为综合入口效率参考）
- 作用：回答「是不是流量入口坏了」「哪个资源位的点击→线索效率在恶化」

**右侧：后端产值效率（线索 → GMV）**
- 各资源位「首单转化率」对比柱状图（3月 vs 4月）
- 各资源位「单线索产出（ARPU）」对比柱状图
- 作用：回答「哪个资源位的线索不值钱了」

**下方：全宽诊断卡片（3 张）**
- 基于真实数据，自动找出变化最大的 3 个环节：
  - 例：「学习页单线索产出：¥84.8 → ¥56.4（↓33%），价格带结构恶化，0元课占比上升」
  - 例：「选课中心 GMV：¥108.4万 → ¥92.3万（↓15%），线索量虽增但转化率与客单价双降」

---

### 屏幕3：根因下钻（产值归因矩阵）

**目标**：支持下钻到「资源位 × 价格带 × 品类类型」三维交叉，定位产值下降的具体病灶。

**布局**：

**模块1：资源位产值矩阵（全宽散点图）**
- 横轴：线索数（对数刻度或线性）
- 纵轴：单线索产出（GMV/线索数）
- 气泡大小：GMV 规模
- 四象限：
  - 右上：高产高值（明星资源位）
  - 左上：低产高值（精准入口，可扩量）
  - 右下：高产低值（流量粗放，需调结构）
  - 左下：低产低值（观察淘汰）
- 点击交互：选中资源位，联动下方模块2/3

**模块2：资源位 GMV 瀑布图（半宽）**
- 展示各资源位 3月→4月 的 GMV 变化量
- 直观看出「谁拖了后腿、谁补了缺口」

**模块3：选中资源位的价格带结构（半宽）**
- 默认展示整体，点击模块1 的某个点后，切换为该资源位的价格带饼图
- 3月 vs 4月 并排环形图
- 标注各价格带占比变化（pp）

**模块4：价格带全局结构（1/3 宽）**
- 3月 vs 4月 双环形图（已有，真实数据）

**模块5：品类类型效率（1/3 宽）**
- 正式品 vs 孵化品 的线索数、转化率、GMV 对比
- 基于 `category_name` 与 CSV `品类` 的映射（已有逻辑）

**模块6：月活用户等级（1/3 宽）**
- 横向堆叠柱状图（已有，真实数据）

---

### 屏幕4：策略建议与行动清单（结论输出）

**目标**：基于屏幕3的下钻结果，自动生成可落地的策略建议，支持一键导出为汇报材料。

**用户明确：不需要「预期 ROI」模拟调配。**

**布局**：

**模块A：核心结论卡片（全宽，3 张）**
- 基于数据自动生成的 3 条核心结论（替代虚构的策略卡片）
- 结论生成逻辑示例：
  - 若某资源位线索量大但单线索产出低于均值 30% → 「XX 资源位线索质量偏低，建议排查价格带结构」
  - 若 0元课占比上升 > 5pp → 「低价引流结构加重，需关注高价值转化」
  - 若某资源位 GMV 环比下降贡献最大 → 「XX 资源位是产值下降主因，需专项复盘」

**模块B：资源位效率预警表（全宽）**
- 列：资源位 | 线索数环比 | 首单转化率环比 | 单线索产出环比 | GMV 贡献变化 | 健康状态
- 健康状态逻辑：
  - 若 CVR 环比下滑 ≥ 10% 或 单线索产出环比下滑 ≥ 15% → 「预警」（黄）
  - 若 GMV 贡献为正且 CVR 稳定 → 「健康」（绿）
  - 其余 → 「观察」（灰）
- 支持点击行下钻到该资源位的历史趋势

**模块C：次月核心行动清单（全宽）**
- 基于模块A 的结论，自动生成行动项（人工可编辑）
- 格式：序号 + 行动标题 + 负责方 + 截止日期 + 优先级
- 数据来源：`action_checklist` 改为由 Python 根据数据自动触发规则生成，而非硬编码
  - 规则示例：若「学习页」单线索产出下降 > 20% → 自动生成「排查学习页价格带结构」

## 需要用户确认的 3 个问题

### 确认1：屏幕1 的 Headline 自动生成逻辑
我提议用「GMV 变化最大的 Top 2 资源位」作为 headline 主因。

示例：
> 4月 GMV 环比下降 10.9%（-¥23.7万），主因：选课中心产值下滑 ¥14.9万 + 学习页产值下滑 ¥10.3万

你是否接受这种归因口径？还是你有其他归因逻辑（如价格带结构、品类类型）需要优先展示？

### 确认2：屏幕3 的「品类类型效率」是否保留？
当前有正式品/孵化品的 CSV 映射逻辑，但用户没明确说过要看这个维度。

问题是：**「正式品 vs 孵化品」的 efficiency 对比，是你们复盘时会讨论的维度吗？**
- 如果是，我保留并做真实数据驱动。
- 如果不是，我删掉旭日图和品类类型模块，避免信息冗余。

### 确认3：屏幕4 的结论生成规则，你是否认可？
我上面写的自动结论规则（基于 CVR 下滑、单线索产出下滑、价格带结构恶化等阈值触发）是**我假设的**。

**你需要告诉我**：
- 你们实际汇报时，策略结论通常从哪些维度得出？
- 是否有固定的复盘 SOP（标准作业流程）？比如「先看资源位排名 → 再看价格带 → 再看品类」？
- 策略卡片里的「产研/运营/内容」分类，是否符合你们实际的责任划分？
